# BM25 搜索 — 代理内置实现

零依赖实现。使用 SQLite FTS5（macOS/Linux/Windows 系统自带），无需安装任何 SDK。

## 使用方式

代理在搜索时执行：

```bash
bash references/bm25-search.sh "transformer attention"
```

或直接用 SQL：

```bash
sqlite3 "$WIKI_PATH/.wiki-index.db" "SELECT path, rank FROM wiki_fts WHERE wiki_fts MATCH 'transformer attention' ORDER BY rank LIMIT 10;"
```

## 初始化索引

首次使用或 wiki 内容变化后，重建索引：

```bash
bash references/bm25-search.sh --reindex
```

## 实现

见 `references/bm25-search.sh` — 纯 bash + sqlite3，零外部依赖。

## BM25 算法简介

BM25（Best Matching 25）是信息检索领域的经典排序算法，Elasticsearch 的默认相关性算法。

SQLite FTS5 内置 BM25 排名函数，通过 `ORDER BY rank` 自动按相关性排序。
rank 值越小（负数）越相关。

**为什么适合 wiki：**
- 词频饱和 — 避免长文档霸榜
- 文档长度归一化 — 短页面和长页面公平竞争
- 逆文档频率 — 罕见词权重更高
- 无需训练 — 纯统计方法
