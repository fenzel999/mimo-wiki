# BM25 搜索 — 代理内置实现

不依赖 MCP 或外部服务。代理直接用 Python 在 wiki 目录上执行 BM25 搜索。

## 依赖

```bash
pip install rank_bm25 jieba
```

- `rank_bm25` — BM25 算法实现
- `jieba` — 中文分词（英文用空格分词即可，中文需要 jieba）

## 搜索脚本

代理执行此脚本搜索 wiki：

```python
import os, sys, re
from rank_bm25 import BM25Okapi
import jieba

wiki_path = os.environ.get("WIKI_PATH", os.path.expanduser("~/wiki"))
query = sys.argv[1] if len(sys.argv) > 1 else exit("用法: python bm25_search.py '查询内容'")

# 收集所有 wiki 页面（排除 raw/ 和 .obsidian/）
docs = {}
for root, dirs, files in os.walk(wiki_path):
    dirs[:] = [d for d in dirs if d not in ("raw", ".obsidian", "_archive", ".git")]
    for f in files:
        if f.endswith(".md"):
            path = os.path.join(root, f)
            rel = os.path.relpath(path, wiki_path)
            with open(path, encoding="utf-8") as fh:
                docs[rel] = fh.read()

if not docs:
    print("wiki 中没有找到 .md 文件")
    sys.exit(1)

# 分词
def tokenize(text):
    # 去掉 frontmatter 和 wikilink 语法
    text = re.sub(r"^---.*?---", "", text, flags=re.DOTALL)
    text = re.sub(r"\[\[([^\]|]*?)(?:\|[^\]]*?)?\]\]", r"\1", text)
    text = re.sub(r"\^[^\s]+", "", text)  # 去掉引用标记
    # 中英文混合分词
    tokens = list(jieba.cut(text.lower()))
    return [t.strip() for t in tokens if len(t.strip()) > 1]

paths = list(docs.keys())
corpus = [tokenize(docs[p]) for p in paths]
query_tokens = tokenize(query)

# BM25 搜索
bm25 = BM25Okapi(corpus)
scores = bm25.get_scores(query_tokens)

# 按分数排序，取 top 10
ranked = sorted(zip(paths, scores), key=lambda x: -x[1])[:10]

print(f"查询: {query}")
print(f"结果: {len(ranked)} 个页面\n")
for i, (path, score) in enumerate(ranked, 1):
    if score > 0:
        # 从 frontmatter 提取标题
        content = docs[path]
        title_match = re.search(r"^title:\s*(.+)$", content, re.MULTILINE)
        title = title_match.group(1).strip() if title_match else path
        print(f"  {i}. [{score:.2f}] {path}")
        print(f"     {title}")
```

## BM25 算法简介

BM25（Best Matching 25）是信息检索领域的经典排序算法，Elasticsearch 的默认相关性算法。

**公式：**
```
score(D, Q) = Σ IDF(qi) · (f(qi, D) · (k1 + 1)) / (f(qi, D) + k1 · (1 - b + b · |D| / avgdl))
```

**为什么适合 wiki：**
- 词频饱和 — 避免长文档霸榜
- 文档长度归一化 — 短页面和长页面公平竞争
- 逆文档频率 — 罕见词权重更高
- 无需训练 — 纯统计方法

详细参数说明见 `rank_bm25` 文档。
