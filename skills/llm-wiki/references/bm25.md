# BM25 搜索 — 代理内置实现

零外部依赖。使用纯 Node.js 实现，只需系统已安装 Node.js（macOS/Linux/Windows 常见开发环境均有）。

## 使用方式

```bash
node references/bm25-search.js "查询内容"     # 搜索
node references/bm25-search.js --reindex      # 重建索引
```

首次搜索时自动建索引。wiki 内容变化后执行 `--reindex` 更新。

## 输出示例

```
查询: transformer attention
结果: 3 个页面

  1. [12.34] concepts/self-attention.md
     Self-Attention (concept)
  2. [8.76] concepts/multi-head-attention.md
     Multi-Head Attention (concept)
  3. [5.12] entities/transformer.md
     Transformer (entity)
```

## 实现细节

- 纯 Node.js，无 npm 依赖
- 中英文混合分词：英文按空格，中文 bigram + unigram
- 自动解析 frontmatter 提取 title/kind/tags
- BM25 参数：k1=1.5, b=0.75
- 索引保存为 `.wiki-index.json`（在 wiki 目录下）

## BM25 算法简介

BM25（Best Matching 25）是信息检索领域的经典排序算法，Elasticsearch 的默认相关性算法。

**为什么适合 wiki：**
- 词频饱和 — 避免长文档霸榜
- 文档长度归一化 — 短页面和长页面公平竞争
- 逆文档频率 — 罕见词权重更高
- 无需训练 — 纯统计方法

## 何时用哪种搜索

| 场景 | 推荐方式 |
|------|---------|
| < 100 页面 | 读 index.md + `search_files` 关键词 |
| 100+ 页面 | BM25 搜索（此脚本） |
| 按 frontmatter 过滤 | search_files + grep |
