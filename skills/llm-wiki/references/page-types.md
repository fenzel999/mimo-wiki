# 页面类型详解

## concept（概念）

独立的想法、技术或模式。这是默认类型，编译的最常见输出。
解释"它是什么"并链接到相关概念。

**回答的问题：** "这个东西是什么？它怎么工作？和其他东西什么关系？"

**内容结构：** 原理 + 变体 + 应用

**示例：** `self-attention`、`knowledge-compilation`、`incremental-compilation`

**创建条件：** 某个技术/想法被多个来源提到，或者是某个来源的核心主题。

## entity（实体）

具体的命名事物：人物、组织、产品、模型。关于特定实例而非抽象模式。
携带唯一标识信息，通常向外链接到其实例化的概念。

**回答的问题：** "这个具体东西的背景、关键信息、相关项目是什么？"

**内容结构：** 元数据 + 背景

**示例：** `andrej-karpathy`、`gpt-4`、`anthropic`、`attention-is-all-you-need`

**创建条件：** 某个具体事物被多个来源提到，需要记录其元数据和背景。

## comparison（比较）

两个或多个概念/实体的并排分析，沿共享维度比较。
链接到所比较的每个主题，使用平行结构。

**回答的问题：** "A 和 B 哪个更好？在什么场景下用哪个？"

**内容结构：** 并排对比表

**示例：** `rag-vs-llmwiki`、`transformer-vs-rnn`、`bm25-vs-dense-retrieval`

**创建条件：** 用户问"A 和 B 的区别"，且答案需要多维度对比。

## overview（总览）

连接某个领域多个相关概念的地图页面。通常由 SCHEMA.md 种子生成，
而非直接从来源提取。提供主题集群入口，向外链接到属于该领域的概念和实体页面。

**回答的问题：** "这个领域有哪些关键概念？从哪里开始？"

**内容结构：** 地图 + 链接

**示例：** `retrieval-augmented-generation-overview`、`attention-mechanisms-overview`

**创建条件：** wiki 在某个领域积累了足够多的概念页面，需要一个入口来组织它们。

## 四种类型对比

| | concept | entity | comparison | overview |
|--|---------|--------|------------|----------|
| **回答什么** | 这是什么？ | 这个具体东西？ | A vs B？ | 这个领域有什么？ |
| **内容结构** | 原理 + 变体 + 应用 | 元数据 + 背景 | 并排对比表 | 地图 + 链接 |
| **更新频率** | 中（技术演变） | 低（事实稳定） | 低（对比结论稳定） | 高（领域不断扩展） |
| **来源要求** | 2+ 来源或核心主题 | 2+ 来源提到 | 用户提问触发 | 领域积累到一定规模 |
